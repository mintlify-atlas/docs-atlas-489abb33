This guide walks you through setting up the TravelConnect backend (FastAPI) and frontend (React/Vite) on your machine, then takes you through your first actions in the app.

## Prerequisites

Before you begin, make sure you have the following installed:

| Tool | Minimum version |
|------|----------------|
| Python | 3.10+ |
| pip | 22+ |
| Node.js | 18+ |
| npm | 9+ |
| MySQL | 8.0+ |

<Tip>
  On macOS, install Python and Node.js via [Homebrew](https://brew.sh). On Windows, use the official installers from [python.org](https://python.org) and [nodejs.org](https://nodejs.org).
</Tip>

## Set up TravelConnect

<Steps>
  <Step title="Clone the repository">
    <CodeGroup>
      ```bash HTTPS
      git clone https://github.com/your-org/travelconnect.git
      cd travelconnect
      ```

      ```bash SSH
      git clone git@github.com:your-org/travelconnect.git
      cd travelconnect
      ```
    </CodeGroup>
  </Step>

  <Step title="Create and configure the database">
    TravelConnect uses a MySQL database. Create the schema before starting the backend:

    ```sql
    CREATE DATABASE travelconect CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
    ```

    The default connection string in `Backend/db/database.py` expects a local MySQL instance with these credentials:

    ```
    Host:     localhost
    Port:     3306
    Database: travelconect
    User:     root
    Password: 1234
    ```

    <Warning>
      Change the default credentials before deploying to any shared or production environment. Update `DATABASE_URL` in `Backend/db/database.py` to match your MySQL configuration.
    </Warning>
  </Step>

  <Step title="Install backend dependencies">
    From the `Backend` directory, install the required Python packages:

    <CodeGroup>
      ```bash pip
      cd Backend
      pip install fastapi uvicorn sqlalchemy pymysql python-jose passlib bcrypt python-multipart
      ```

      ```bash pip (virtual environment)
      cd Backend
      python -m venv .venv
      source .venv/bin/activate   # Windows: .venv\Scripts\activate
      pip install fastapi uvicorn sqlalchemy pymysql python-jose passlib bcrypt python-multipart
      ```
    </CodeGroup>

    <Note>
      A `requirements.txt` is not included in the repository. The packages listed above correspond to all imports found across the backend source files.
    </Note>
  </Step>

  <Step title="Start the FastAPI server">
    From the `Backend` directory, start the API with `uvicorn`:

    ```bash
    uvicorn main:app --reload
    ```

    The server starts on **http://localhost:8000**. On first run, SQLAlchemy automatically creates all database tables via `Base.metadata.create_all`.

    Verify the API is running:

    ```bash
    curl http://localhost:8000/
    # {"mensaje":"HOLA"}
    ```

    You can also browse the auto-generated interactive API docs at **http://localhost:8000/docs**.
  </Step>

  <Step title="Install frontend dependencies">
    Open a new terminal, then from the `FrontEnd` directory:

    ```bash
    cd FrontEnd
    npm install
    ```
  </Step>

  <Step title="Start the Vite dev server">
    ```bash
    npm run dev
    ```

    The app is available at **http://localhost:5173**. The frontend is pre-configured to send API requests to `http://localhost:8000`, so both servers must be running at the same time.
  </Step>
</Steps>

## First steps

With both servers running, try these actions to confirm everything is working end-to-end.

<Steps>
  <Step title="Register an account">
    Navigate to **http://localhost:5173/register** and fill in your email, full name, and password.

    New accounts are assigned the **Viatger** (Traveler) role by default. This role lets you browse trips and enroll as a participant.

    You can also register directly through the API:

    ```bash
    curl -X POST http://localhost:8000/auth/register \
      -H "Content-Type: application/json" \
      -d '{"email": "you@example.com", "full_name": "Your Name", "password": "yourpassword"}'
    ```
  </Step>

  <Step title="Log in">
    Go to **http://localhost:5173/login** and sign in with the credentials you created. After a successful login, the app stores your JWT in `localStorage` and attaches it automatically to every subsequent API request via the `Authorization: Bearer` header.
  </Step>

  <Step title="Browse trips">
    After logging in, you are redirected to the dashboard at **/dashboard**, which lists all available trips. Click any trip to open its detail page at `/trips/:id`, where you can view the destination, itinerary, and current participants.
  </Step>

  <Step title="Enroll in a trip">
    On a trip detail page, click **Enroll** to join the trip. This sends a `POST` request to `/viatge/trips/:id/enroll` and adds you as a participant.

    To leave a trip you have already joined, click **Leave**, which calls `/viatge/trips/:id/leave`.

    <Note>
      Only users with the **Viatger** role can enroll in trips. Creators and admins manage trips but do not enroll as participants.
    </Note>
  </Step>
</Steps>

## Available scripts

Run these commands from the `FrontEnd` directory:

| Command | Description |
|---------|-------------|
| `npm run dev` | Start the Vite development server on port 5173 |
| `npm run build` | Build the frontend for production |
| `npm run preview` | Preview the production build locally |
| `npm run lint` | Run ESLint across the frontend source |

## User roles

TravelConnect has three roles. A user's role is assigned at registration or updated by an admin.

| Role | Access |
|------|--------|
| `Viatger` | Browse trips, enroll/leave trips, request a promotion to Creador |
| `Creador` | All Viatger access, plus create and edit trips |
| `Admin` | Full access, including user management at `/admin/users` and promotion request review at `/admin/requests` |

To request a promotion from Viatger to Creador, navigate to **/promotion** after logging in.
