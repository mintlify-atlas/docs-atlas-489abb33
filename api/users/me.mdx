---
title: "Get Current User"
description: "Retrieve the profile of the currently authenticated user."
---

## Endpoint

`GET /users/users/me`

Returns the full profile of the user identified by the supplied Bearer token.

<Note>
  The double `/users/users/` segment is intentional: the router is mounted at `/users` and defines an internal prefix of `/users`, so the combined path is `/users/users/me`.
</Note>

## Authentication

Required. Send a valid Bearer token in the `Authorization` header.

## Request

No request body or query parameters.

## Response

<ResponseField name="id" type="integer">
  The user's unique identifier.
</ResponseField>

<ResponseField name="email" type="string">
  The user's email address.
</ResponseField>

<ResponseField name="full_name" type="string">
  The user's display name, or `null` if not set.
</ResponseField>

<ResponseField name="rol" type="string">
  The user's role: `Viatger`, `Creador`, or `Admin`.
</ResponseField>

<ResponseField name="bio" type="string">
  The user's biography, or `null` if not set.
</ResponseField>

## Example

```bash
curl http://localhost:8000/users/users/me \
  -H "Authorization: Bearer <access_token>"
```

```json
{
  "id": 1,
  "email": "alice@example.com",
  "full_name": "Alice",
  "rol": "Viatger",
  "bio": "Travel enthusiast based in Barcelona."
}
```

## Errors

| Status | Detail | Description |
|--------|--------|-------------|
| `401 Unauthorized` | `Token no vàlid` | The token is missing, malformed, or expired. |
| `401 Unauthorized` | `Token sense ID` | The token does not contain a user ID (`sub` claim). |
| `404 Not Found` | `Usuari no trobat` | The user ID in the token no longer exists in the database. |
