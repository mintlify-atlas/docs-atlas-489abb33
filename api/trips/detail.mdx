---
title: "Get Trip"
description: "Retrieve a single trip by its ID."
---

## Endpoint

`GET /viatge/trips/{id}`

Returns the full details of a single trip identified by its numeric ID.

## Authentication

Not required.

## Request

<ParamField path="id" type="integer" required>
  The unique identifier of the trip.
</ParamField>

## Response

<ResponseField name="id" type="integer">
  The trip's unique identifier.
</ResponseField>

<ResponseField name="nom" type="string">
  The trip name.
</ResponseField>

<ResponseField name="desti" type="string">
  The destination of the trip.
</ResponseField>

<ResponseField name="data_inici" type="string">
  Start date in `YYYY-MM-DD` format.
</ResponseField>

<ResponseField name="data_fi" type="string">
  End date in `YYYY-MM-DD` format.
</ResponseField>

<ResponseField name="descripcio" type="string">
  Optional description of the trip.
</ResponseField>

<ResponseField name="estat" type="string">
  Planning status: `Planificant`, `Actiu`, `Complet`, or `Cancel·lat`.
</ResponseField>

<ResponseField name="creador_id" type="integer">
  ID of the user who created the trip.
</ResponseField>

<ResponseField name="maxim_participants" type="integer">
  Maximum number of participants allowed.
</ResponseField>

## Example

```bash
curl http://localhost:8000/viatge/trips/1
```

```json
{
  "id": 1,
  "nom": "Ruta pels Pirineus",
  "desti": "Pirineus",
  "data_inici": "2026-07-01",
  "data_fi": "2026-07-10",
  "descripcio": "Excursió de 10 dies pels Pirineus catalans.",
  "estat": "Planificant",
  "creador_id": 2,
  "maxim_participants": 12
}
```

## Errors

| Status | Detail | Description |
|--------|--------|-------------|
| `404 Not Found` | _(varies)_ | No trip exists with the given ID. |
