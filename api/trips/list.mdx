---
title: "List Trips"
description: "Retrieve all available trips on the platform."
---

## Endpoint

`GET /viatge/trips`

Returns a list of all trips. No authentication is required, making this suitable for public-facing pages.

## Authentication

Not required.

## Request

No parameters.

## Response

An array of trip objects.

<ResponseField name="id" type="integer">
  The trip's unique identifier.
</ResponseField>

<ResponseField name="nom" type="string">
  The trip name.
</ResponseField>

<ResponseField name="desti" type="string">
  The destination of the trip.
</ResponseField>

<ResponseField name="data_inici" type="string">
  Start date in `YYYY-MM-DD` format.
</ResponseField>

<ResponseField name="data_fi" type="string">
  End date in `YYYY-MM-DD` format.
</ResponseField>

<ResponseField name="descripcio" type="string">
  Optional description of the trip.
</ResponseField>

<ResponseField name="estat" type="string">
  Planning status: `Planificant`, `Actiu`, `Complet`, or `Cancel·lat`.
</ResponseField>

<ResponseField name="creador_id" type="integer">
  ID of the user who created the trip.
</ResponseField>

<ResponseField name="maxim_participants" type="integer">
  Maximum number of participants allowed.
</ResponseField>

## Example

```bash
curl http://localhost:8000/viatge/trips
```

```json
[
  {
    "id": 1,

    "data_fi": "2026-07-10",
    "descripcio": "Excursió de 10 dies pels Pirineus catalans.",
    "estat": "Planificant",
    "creador_id": 2,
    "maxim_participants": 12
  }

```

## Errors

This endpoint does not return application-level errors under normal circumstances.
